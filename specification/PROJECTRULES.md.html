项目文件夹命名规范

项目中的所有文件夹命名方法均使用大驼峰式命名方法。

项目文件命名规范

项目中所有的文件命名均以驼峰式命名方式进行命名，只有当特殊情况下，组件需要单独拆离出 index.tsx，这时候文件的命名需要用大写字母开头。

公共和私有组件以及布局命名规则

组件或布局将在多个组件中使用，例如页面布局或页面。 通用命名约定将是根据组件或布局的 ui 的名字，然后紧跟其功能的名字，依此类推，例如，有一个button将从数据库fetch``userData, 我们可以将其命名为ButtonFetchUserData，同时，您需要确保组件文件夹和组件名称相同。

CSS 中 class 和 id 命名规范

通常，我们会使用 BEM 命名约定来命名项目 CSS 类和 ID。 但是，因为项目中的所有类和 ID 名称都已被哈希处理。 我们将使用以下方法在项目中命名类和 id

Class/id 的firstname是当前组建的名称，随后我们需要为该 class/id 添加element和modifiers，我们会通过一个下划线来连接firstname和element，我们会通过两个下划线来将modifiers和其他进行连接。

如果我们有一个组件，并且想要将组件的header部分从活动状态更改为非活动状态，则当此组件处于非活动状态时，我们需要向该组件添加一个的 class。 因此，此类名称将为component_header__inactive，同时 class 和 id 的命名均以小写字母开头。

CSS 中 rem 的强制使用

在项目上线后，我们需要根据不同的屏幕尺寸来对前端的尺寸进行等比缩放，这时候就需要通过调节 html 中的 font-size 来进行等比缩放，这一缩放机制是通过 rem 来实现的，所以在开发的过程中禁止使用px做为单位，任何时候都需要以rem做为尺寸单位。项目中rem和px的换算单位为1rem 等于10px。

项目中图片命名规范

在项目中引入的图片需统一命名后放在src/Assets/images文件夹下，命名规则为图片功能类别__图片名称

例如：icon_userAvata 则代表着用户的头像图表， logo_menu 则代表着在菜单栏中所用到的 logo 图标，spr_blogCard 则代表着博客页面中卡片元素所使用的图片。

  • 图片功能类别

    图片功能类别 含义
    icon 项目中 fontawesome 没有包含的 icon
    logo Logo 类图片
    bg 可平铺或大背景图
    btn 按钮
    spr 单页面中各种其他图片命名集合

  • 图片名称 图片名称需要根据图片使用的场景命名，例如用户头像：userAvatar， 商品列表：productList 等。

变量，常量，函数和类的命名规范

项目中对于变量，常量，函数和类的命名极其重要，这可以快速帮助你团队中的成员了解到你所写的代码的含义。

  • 变量

    命名方法：小驼峰命名法。
    命名规范：前缀为名词

    let maxCount = 10;

  • 常量 命名方法：小驼峰命名法。
    命名规范：前缀为名词

    const maxCount = 10;

  • 函数 命名方法：小驼峰命名法。
    命名规范：前缀应当为动词
    命名建议前缀：

    动词 含义 返回值
    handle 处理 event 事件的函数 可返回任意值或者无返回值
    is 判断真假 返回一个布尔值
    get 获取某个值 返回一个非布尔值
    set 设置某个值 无返回值
    load 加载某些参数 无返回值或者返回是否加载完成的结果
    remove 移除某些样式或移除某些参数 可返回任意值或无返回值
    add 添加某些样式或者添加某些参数 可返回任意值或无返回值
    change 修改某些样式或者修改某些参数 可返回任意值或无返回值
    watch 监听某一函数或事件 可返回任意值或无返回值
    dispatch 发送某一 action 返回一个对象

  • 类 命名方法：大驼峰命名法
    命名规范：前缀为类的名称

    class Person {
public name: string;
constructor(name) {
  this.name = name;
}
}
const person = new Person('弗兰克');

代码规范注释

顶层/文件注释

顶级注释用于告诉不熟悉此代码的读者此文件中包含的内容。 应提供文件的一般内容，其作者，依赖关系和其他相关信息。

/**
 * @file 文件的描述
 * @date 2020/06/25 13:11
 * @author 作者的名字
 * @lastModify 作者的名字 2020/06/25 14:11
 */

方法与函数的注释

@param 标签提供了函数参数的各种描述，包括参数名称，参数数据类型和描述。 @param {变量类型}变量名-变量描述
例如:
@params {string | Number}
@callback 回调函数的描述.
@returns 函数 return 值的描述。

/**
 * 发送一个post请求
 * @param {string} url - 请求的 url
 * @param {string} method - 请求的方法
 * @param {Object} body - 请求的body
 * @callback successCallBack-requestSuccessCallBack - 请求成功的回调函数
 * @callback errorCallBack-requestErrorCallBack - 请求失败的回调函数
 * @returns {Promise.<*>} - 返回值
 */
const requestUrl = async (url, method, body, successCallBack, errorCallBack) => {
    reurn new Promise...
};

变量与属性的注释

@type 用于标识标识符可能包含的值的类型或函数返回的值的类型。

/**
 * 变量描述
 * @let
 * @type {number)}
 */
let foo = 1;

定量注释

/**
 * 定量描述
 * @constant
 * @type {string}
 *
 */
const RED = "FF0000";
